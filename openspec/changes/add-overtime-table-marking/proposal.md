## Why

賽事中部分桌次需要加時處理（+3/+6/+9 分鐘或自訂時間），裁判需要快速識別哪些桌次有加時，現有系統無法標記此資訊，導致需依靠外部筆記輔助，增加作業負擔。

## What Changes

- 桌號圖卡支援長按進入加時標記模式，顯示快選按鈕（+3、+6、+9）及手動輸入欄位
- 加時標記獨立於桌號狀態（尚未結束／已分配／已完成），不影響狀態循環
- 已標記加時的圖卡上顯示加時數值徽章
- 長按已有加時標記的圖卡可修改或清除加時標記
- 主畫面最上方新增「只顯示加時桌」切換按鈕，啟用時畫面僅顯示有加時標記的桌號
- 「只顯示加時桌」與「顯示已完成桌號」checkbox 為獨立篩選條件，同時套用時取交集（例：啟用只顯示加時桌 + 取消勾選已完成 → 僅顯示未完成的加時桌）

## Capabilities

### New Capabilities
- `overtime-table-marking`: 加時標記的新增、修改、清除，以及圖卡上的顯示
- `overtime-filter-view`: 只顯示加時桌的切換按鈕與篩選邏輯（與現有已完成篩選獨立並可疊加）

### Modified Capabilities
- `eor-tracker-core`: 桌號圖卡的可見性規則新增加時篩選條件（原有條件為已完成顯示控制，需擴充）

## Impact

- `app.js`：桌號資料結構新增 `overtimeMinutes` 欄位；新增長按事件處理；修改 `renderTracker()` 篩選邏輯；新增加時篩選狀態
- `index.html`：新增「只顯示加時桌」按鈕；新增加時標記輸入 UI（彈出層或 inline）
- `style.css`：新增加時徽章樣式；新增加時標記輸入介面樣式；新增按鈕啟用狀態樣式
